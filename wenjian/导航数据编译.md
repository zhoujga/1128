# 导航数据编译

### 1. 技术方案总览
针对特种终端资源受限、需离线导航的特点，提出一种基于**分级网格化数据组织**与**正规化坐标压缩存储**的导航数据编译方法。通过先进行逻辑组织规划，再映射为高效的 SQLite 物理数据库结构，实现高性能的空间查询与路径规划。

### 2. 数据组织设计（逻辑模型）

#### 2.1 分级网格体系与编码规则
采用两级网格体系以平衡查询效率与数据量，网格 ID（RegionID）遵循严谨的编码公式：
*   **网格划分：**
    *   **一次网格：** 经差 1°、纬差 40′。
    *   **二次网格：** 在一次网格基础上细分为 8×8，每格经差 7.5′、纬差 5′。
*   **计算公式：**
    *   $RR = \text{floor}(\text{纬度} \times 60 / 40)$
    *   $LL = \text{floor}(\text{经度}) - 60$
    *   $M, N$：二次网格在一次网格内的行、列号（0-7）。
    *   网格编码 = $RR \times 10000 + LL \times 100 + M \times 10 + N$。

#### 2.2 动态网格选择规则
*   **触发条件：** 统计每个一次网格内的道路元素（Road Element）数量。
*   **处理机制：** 若道路数量 $\ge 2048$，则自动触发二次网格分裂，将数据分散存储于对应的 64 个二次网格中。
*   **目的：** 确保单个数据块（Page/Block）大小适中，避免终端内存溢出。

#### 2.3 正规化坐标体系 (Normalized Coordinates)
为了在保持高精度的同时压缩存储空间，网格内坐标采用 16 位无符号整数存储：
*   **逻辑：** 存储点位相对于所属网格左下角参考点（Reference Point）的偏移比例。
*   **精度：** 将网格跨度等分为 65536 份。对于二次网格，其定位精度可达 1.2 米级，满足特种机动需求。

### 3. 数据库结构设计（物理模型）

#### 3.1 网格信息表 (grid_info)
存储网格元数据及绝对位置基准。
```sql
CREATE TABLE grid_info (
    grid_id        INTEGER PRIMARY KEY,  -- 遵循编码规则
    level          TINYINT,              -- 1:一次网格, 2:二次网格
    ref_lon        REAL,                 -- 网格参考点经度 (WGS84)
    ref_lat        REAL,                 -- 网格参考点纬度
    road_count     INTEGER,              -- 道路要素总数
    node_count     INTEGER               -- 结点总数
);
```

#### 3.2 结点表 (node)
```sql
CREATE TABLE node (
    node_id        INTEGER PRIMARY KEY,
    grid_id        INTEGER,              
    norm_x         INTEGER,              -- 正规化坐标 X (0-65535)
    norm_y         INTEGER,              -- 正规化坐标 Y (0-65535)
    boundary_flag  TINYINT,              -- 是否为跨网格边界点
    link_ids       BLOB                  -- 关联弧段 ID 序列 (Varint 压缩存储)
);
```

#### 3.3 弧段表 (link) - 特种增强版
增加针对越野机动和通过性限制的专业属性。
```sql
CREATE TABLE link (
    link_id        INTEGER PRIMARY KEY,
    grid_id        INTEGER,
    net_class      TINYINT,              -- 道路等级 (用于虚拟分层加载)
    start_node     INTEGER,
    end_node       INTEGER,
    direction      TINYINT,              -- 通行方向
    -- 特种约束属性
    max_weight     REAL,                 -- 限重 (吨)
    max_height     REAL,                 -- 限高 (米)
    soil_type      TINYINT,              -- 土质分级 (用于越野机动性评估)
    risk_level     TINYINT,              -- 区域风险系数
    geometry       BLOB                  -- 形状点序列 (正规化坐标流压缩存储)
);
```

### 4. 编译流程

#### 4.1 数据预处理与归一化
*   统一坐标系为 WGS84。
*   将地理坐标（REAL）根据网格基准转换为 16 位正规化坐标（INTEGER）。

#### 4.2 拓扑修复与边界结点处理
*   **跨网格打断：** 当道路跨越网格边界时，在交界处强制插入“边界结点”。
*   **同名关联：** 确保相邻网格共享相同的边界结点 ID，实现逻辑闭合。

#### 4.3 物理存储优化（虚拟分层）
*   **排序重排：** 每个网格内的弧段 ID 按照 `net_class`（道路等级）进行物理排序。
*   **优势：** 终端搜索时可优先加载高等级道路数据，实现“大范围搜索、小数据加载”。

#### 4.4 索引构建与性能调优
*   建立 `grid_id` 聚簇索引。
*   对 `link` 表的 `start_node` 和 `end_node` 建立复合索引以加速路径搜索算法（如 A*）。
*   执行 `VACUUM` 和 `ANALYZE` 优化 B-Tree 结构。
