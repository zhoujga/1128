## 发明名称

**一种面向越野机动车的多源地形数据融合的导航数据编译方法**

---

## 所属技术领域

越野机动车导航技术领域、多源地理空间数据融合技术领域、无路区域路径规划技术领域

---

## 背景技术

随着越野探险、野外救援、军事作战等应用场景的需求日益增长，越野机动车导航系统面临着与传统道路导航完全不同的技术挑战。传统高德导航、百度导航等主流导航软件基于城市道路网络设计，依赖完整的道路网数据，无法适用于无路区域或非结构化地形的导航场景。

现有技术主要存在以下问题：（1）数据源依赖问题：传统导航依赖完整道路网数据，无法在没有道路的区域生成导航路径；（2）地形因素考量不足：现有技术未综合考虑坡度、植被、水域、土壤等地形因素对机动车通行的实际影响；（3）空间索引效率低下：大范围地形数据处理时查询效率低，无法满足实时路径规划需求；（4）数据体积庞大：地形数据包含多维度信息，数据量通常达到数GB级别，终端存储和加载困难；（5）技术门槛高：现有导航数据编译方法涉及复杂的二进制数据结构设计、文件偏移量计算等技术，对开发人员专业能力要求较高。

越野机动车在无路区域行驶时面临特殊的环境限制，包括地形复杂多样（山地、沙漠、丛林、沼泽等）、坡度变化剧烈（需要考虑爬坡能力和下坡安全性）、植被覆盖密集（影响视野和通行性）、水域障碍（需要判断涉水可行性）、土壤承载力差异（影响车辆通过性）、设施建筑障碍（需要规避）等因素。

因此，亟需一种面向越野机动车的多源地形数据融合的导航数据编译方法，以解决上述技术问题。

---

## 发明目的

本发明的目的有三个方面：第一，解决无路区域导航数据缺失问题，针对越野机动车在无路区域行驶时缺乏导航数据的问题，提供一种基于多源地形数据融合的导航数据编译方法；第二，实现多维度地形因素的综合考量，通过整合坡度、植被、水域、土壤、建筑等多源栅格数据，建立多维度权重体系，计算机动车在不同地形条件下的通行成本；第三，提高空间数据处理效率，采用H3六边形格网技术进行空间索引，实现高效的空间查询和路径规划。

---

## 技术方案

本发明专利是一种面向越野机动车的多源地形数据融合的导航数据编译方法，该方法通过整合多源栅格地形数据、构建H3六边形格网体系、计算多维度通行权重、生成可达路网，实现越野环境下的路径规划。

图1为技术流程图，下面将详细列出关键步骤：

```
栅格数据整合 → H3格网生成 → 中心点选取 → 权重计算 → 路径规划 → 层级处理
```

（一）多源栅格数据采集与整合

本发明采用多源数据整合方法，数据采集来源包括基础地理数据（数字高程模型DEM、坡度数据slope.tif、土地覆盖数据）、专项地形数据（植被覆盖数据vegetation.tif包含草地grass、灌木shrub、丛林jungle、隐蔽地take_cover，水域数据river_system.tif，居民地数据residential_area.tif，设施建筑数据facility.tif，土壤数据soil.tif包含岩石土壤rocky_soil、沙质土壤sandy_soil、沙壤土Sandy_loam_soil、沙质黏土sandy_clay、黏土clay、泥炭土peat_soil、盐碱土salinized_soil，地貌数据landforms）以及范围边界数据road.sqlite数据库（表名：outline）。数据处理步骤包括坐标统一（所有空间数据转换至WGS84坐标系EPSG:4326）、分辨率统一（栅格数据统一为10m分辨率）、格式转换（将矢量数据栅格化为TIFF格式）。

（二）H3六边形格网生成

本发明采用Uber H3六边形格网技术进行空间划分，与传统经纬度网格相比，H3格网具有全球统一的格网编号体系、近似均匀的格网面积、高效的邻域查询算法、多级别分辨率支持等优势。本发明采用两级格网体系，父格网为Level 10覆盖约100km²用于区域划分和路径规划约束，子格网为Level 12覆盖约3.5km²用于精细导航和权重计算。格网生成方法包括：首先从SQLite数据库读取研究区域边界outline图层；然后使用H3库的polygonToCells函数将边界多边形转换为H3格网编码，关键实现代码如下：
```cpp
H3_EXPORT(maxPolygonToCellsSize)(&polygon, res, 0, &polygonToCellsSize);
H3Index *polygonToCellsOut = (H3Index*)calloc(polygonToCellsSize, sizeof(H3Index));
H3_EXPORT(polygonToCells)(&polygon, res, 0, polygonToCellsOut);
for (int i = 0; i < polygonToCellsSize; i++) {
    if (polygonToCellsOut[i] == 0) continue;
    cells.emplace(polygonToCellsOut[i]);
}
```
接着遍历每个格网提取该格网区域内的多维度地形属性（坡度、植被、水域、土壤、建筑等）；最后生成Shapefile格式的格网数据，包含格网编码和属性信息。

（三）中心点选取与可达性分析

在每个父格网（Level 10）中选取一个子格网（Level 12）作为代表中心点，选取规则包括四个步骤：首先进行边界格网识别，识别位于父格网边界的子格网（与不同父格网相邻），关键实现代码如下：
```cpp
H3_EXPORT(gridRingUnsafe)(child_cell, 1, child_boundarycell_index);
for (int v = 0; v < 6; v++) {
    H3_EXPORT(cellToParent)(child_boundarycell_index[v], parentCellLevel, &parent_index);
    if (parent_index != parent_cell) {
        isBoundaryCell = true;
    }
}
```
其次进行可达性格筛除，筛除不可通行的格网（坡度过陡如大于30度、建筑占据、水域阻隔），关键实现代码如下：
```cpp
if (smallCellsIter->second.slope < accessSlope && 
    smallCellsIter->second.building < accessBuilding) {
    accessibleCells.emplace(child_cells[i]);
}
```
然后进行连通性验证，使用广度优先搜索（BFS）验证中心点与相邻父格网的连通性，关键实现代码如下：
```cpp
bool checkConnectivity(H3Index& startCellIndex, 
    std::unordered_set<H3Index>& accessibleCells, 
    std::unordered_set<H3Index>& availablePath);
```
最后进行几何中心计算，对连通区域内的格网点计算几何中心，关键实现代码如下：
```cpp
LatLng geo;
double lat_sum = 0.0, lng_sum = 0.0;
for (auto cell : availablePath) {
    H3_EXPORT(cellToLatLng)(cell, &geo);
    lat_sum += radsToDegs(geo.lat);
    lng_sum += radsToDegs(geo.lng);
}
H3Index centerPointIndex = selectClosestCell(centerPoint, accessibleCells);
```

（四）多维度通行权重计算

本发明的核心创新之一是建立了综合多维度地形因素的通行权重计算模型。权重系数定义包括坡度slope权重为1.0作为主要通行限制因素，水域water权重为0.4作为涉水难度系数，建筑building权重为0.1作为建筑障碍系数，植被类包括草地grass、灌木shrub、丛林jungle、隐蔽地take_cover权重均为0.1，设施facility权重为0.1作为人造设施障碍，地貌landforms权重为0.1作为地貌类型系数，土壤类包括岩石土壤rocky_soil、沙质土壤sandy_soil、沙壤土Sandy_loam_soil、沙质黏土sandy_clay、黏土clay、泥炭土peat_soil、盐碱土salinized_soil权重均为0.1作为土壤承载力系数。权重计算公式实现代码如下：
```cpp
static double getLinkCost(NodeInfo & mNodeInfo) {
    double sum = 0.0;
    sum = mNodeInfo.building * b_high_effect
        + mNodeInfo.grass * grass_effect
        + mNodeInfo.shrub * shrub_effect
        + mNodeInfo.jungle * jungle_effect
        + mNodeInfo.take_cover * take_cover_effect
        + mNodeInfo.rocky_soil * rocky_soil_effect
        + mNodeInfo.sandy_soil * sandy_soil_effect
        + mNodeInfo.Sandy_loam_soil * Sandy_loam_soil_effect
        + mNodeInfo.sandy_clay * sandy_clay_effect
        + mNodeInfo.clay * clay_effect
        + mNodeInfo.peat_soil * peat_soil_effect
        + mNodeInfo.salinized_soil * salinized_soil_effect
        + abs(mNodeInfo.slope) * slope_effect
        + mNodeInfo.landforms * l_high_effect
        + mNodeInfo.water * water_effect
        + mNodeInfo.facility * facility_effect;
    return sum;
}
```
不可通行判定规则为当坡度大于等于30度或建筑占据时，该格网不可通行，设置权重值为10000.0，实现代码如下：
```cpp
if (mNodeInfo.slope >= accessSlope || mNodeInfo.building >= accessBuilding) {
    weightDatas[cellCode] = 10000.0;  // 不可通行
}
```

（五）基于Dijkstra算法的路径规划

在H3格网上使用Dijkstra算法寻找中心点之间的最优路径，算法实现具有以下特点：首先进行父子格网约束，路径跨越父格网边界时进行限制，防止路径过于绕行，实现代码如下：
```cpp
H3Index parent_cell;
H3_EXPORT(cellToParent)(neighbor, parent_res, &parent_cell);
if (((parent_cell == start_parent_cell || 
     0 == currNode->up_cells.count(parent_cell)) && 
    2 == currNode->up_cells.size())) {
    continue;  // 跳过
}
```
其次进行邻域查询，使用H3的gridRingUnsafe函数快速获取相邻格网，实现代码如下：
```cpp
H3Index neighbors[6] = {0};
H3_EXPORT(gridRingUnsafe)(currNode->cell, k, neighbors);
```
然后进行权重边值计算，边权重为通行成本，累加计算路径总成本。同时进行父格网状态追踪，记录路径经过的父格网集合，用于约束路径规划，实现代码如下：
```cpp
neighbor_node->up_cells.insert(parent_cell);
neighbor_node->up_cells.insert(currNode->up_cells.begin(), 
                                currNode->up_cells.end());
```
路径输出与简化方面，生成的路径包含冗余拐点，需要进行简化处理，包括拐点识别（计算相邻三点的向量叉积，识别方向变化点，实现代码如下：
```cpp
double angle = std::abs((p2.getY() - p1.getY()) * (p3.getX() - p2.getX()) 
                      - (p2.getX() - p1.getX()) * (p3.getY() - p2.getY()));
if (angle > 1e-10) {
    inflection_points.push_back(p2);
}
```
）、冗余点移除（排除首尾点和拐点后的中间点即为冗余点）、简化路径生成（只保留拐点，生成最终路径）。

（六）路网层级处理

使用Boost图论库对生成的路网进行层级抽稀处理，包括图构建（将路网转换为Boost邻接表表示，实现代码如下：
```cpp
typedef boost::adjacency_list<boost::vecS, boost::vecS, 
    boost::undirectedS, boost::no_property, 
    boost::property<boost::edge_weight_t, double>> LevelGraph;
```
）、连通分量分析（识别独立的路网连通区域）、中心性计算（计算各边的重要性指标）、层级抽稀（根据道路重要性保留关键路径，生成不同层级的路网数据）。

（七）数据输出与格式

输出格式包括Shapefile格式用于GIS软件查看和编辑，以及GeoPackage格式用于移动端导航数据。输出字段包括snode（int64类型，起点H3编码）、enode（int64类型，终点H3编码）、level（int类型，路径层级）、cost（double类型，通行成本）、up_slope（double类型，上坡坡度）、down_slope（double类型，下坡坡度）、grass/shrub/jungle/take_cover（int类型，植被类型标记）、water（double类型，水域影响）、building（double类型，建筑影响）、facility（int类型，设施影响）、landforms（int类型，地貌类型）、slope（double类型，坡度值）、rocky_soil/sandy_soil等（int类型，土壤类型标记）。

## 有益效果

本发明针对越野机动车在无路区域行驶时的导航需求，提供一种基于多源地形数据融合的导航数据编译方法。与现有技术相比，本发明具有以下有益效果：第一，解决无路区域导航问题，传统导航依赖道路网数据，无法在无路区域生成导航路径，本发明通过分析地形因素的可达性，在没有道路的区域生成可通行的路径数据，填补了无路区域导航数据的空白；第二，多维度地形综合考量，通过整合16种地形因素（坡度、水域、植被、土壤、建筑等），建立综合权重体系，更准确地反映越野机动车在不同地形条件下的实际通行能力，比传统单一因素分析方法更加科学合理；第三，高效的空间索引，采用H3六边形格网技术进行空间索引，具有查询效率高（O(1)时间复杂度定位格网）、邻域查询快（常数时间获取相邻格网）、多分辨率支持（灵活的空间尺度选择）等优势；第四，数据量适中，相比传统导航数据（数GB级别），本发明生成的路网数据量适中，单个格网数据仅需数百KB，便于移动端存储和加载；第五，技术门槛降低，采用成熟的GDAL、H3、Boost等开源库，开发人员无需关注底层算法实现，降低了导航数据编译的技术门槛。

## 具体实施方式

实施例一为山区越野导航数据生成，步骤包括准备山区DEM数据和土地利用数据、生成H3格网（Level 10/12）、提取各格网的坡度植被水域等属性、选取可达中心点（排除坡度大于30度、建筑占据的区域）、计算格网间通行权重、使用Dijkstra算法生成路径、简化路径去除冗余点、输出Shapefile路网数据。实施例二为沙漠越野导航数据生成，步骤包括准备沙漠区域DEM和土壤数据、生成H3格网、重点考虑沙质土壤承载力和植被覆盖等因素、选取可行中心点、调整权重系数（增加沙质土壤权重）、生成沙漠适用路网。
